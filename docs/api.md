> API 文档是开发者的协作指南，通过清晰定义接口功能、用法，确保团队在开发过程中保持一致。
>
> 本文档需要列出软件后端服务提供的所有 API，并按照合理的结构进行组织（可按模块 / 资源类型划分）。

**API** **文档编写建议**

- 每个接口至少包含**请求方法和接口地址**、**接口说明**、**使用示例**、**请求参数**、**响应状态**、**响应数据**（如有），必要时可包含其他信息。
- 重视**示例**的引导作用，它们可以帮助其他组员快速理解 API 调用方式。
- 给接口起名时，应让关键信息在目录中展示，方便查找。
- 推荐采用以下方案管理 API 文档，提高协作效率：
  - Postman、Apifox 等 API 管理工具
  - 在代码注释中写 API 文档，用生成工具自动生成 API 文档（如 swaggo）
  - 在共享文档中，按照约定格式维护 API 文档（可参考本模板提供的示例）
- 我们**接受多种形式**提交 API 文档，包括但不限于以下形式：
  - 飞书文档 / 腾讯文档
  - API 管理工具的分享链接
  - HTML（如 Swagger / mkdocs）
  - PDF / Word

## 业务状态码约定

后端返回的 json 均为以下格式

| **参数** | **类型** | **说明**       |
| -------- | -------- | -------------- |
| code     | int      | 业务状态码     |
| message  | str      | 可读的错误消息 |
| data     | dict     | 响应数据       |

业务状态码分段：

| **业务状态码** | **说明**                             |
| -------------- | ------------------------------------ |
| 0              | 成功                                 |
| 10***          | 通用错误（参数格式错误、请求频繁……） |
| 20***          | 用户系统                             |
| 30***          | 日程管理系统                         |

## HTTP 状态码

见 [RESTful](https://restfulapi.net/http-status-codes/) 文档。

## 用户系统

### 用户注册

**[POST]**  `/auth/register`

用户使用邮箱、用户名和密码注册账号。

**请求参数**

| **参数** | **类型** | **说明** |
| -------- | -------- | -------- |
| email    | str      | 邮箱     |
| username | str      | 用户名   |
| password | str      | 密码     |

**响应状态**

| **状态码** | **说明**       |
| ---------- | -------------- |
| 0          | 注册成功       |
| 10001      | 请求参数错误   |
| 20001      | 用户名已被占用 |

**响应数据**

| **参数** | **类型** | **说明** |
| -------- | -------- | -------- |
| user_id  | int      | 用户编号 |
| username | str      | 用户名   |

**示例**

请求

```JSON
{
    "email": "tom@gmail.com",
    "username": "Tom",
    "password": "******" // hashed password
}
```

响应

```JSON
{
    "code": 0,
    "message": "注册成功",
    "data": {
        "user_id": 1,
        "username": "Tom"
    }
}
```

**响应数据**

| **参数** | **类型** | **说明** |
| -------- | -------- | -------- |
| user_id  | int      | 用户编号 |
| username | str      | 用户名   |

**示例**

请求

```JSON
{
    "username": "Tom",

}
```

响应

```JSON
{
    "code": 0,
    "message": "登录成功",
    "data": {
        "user_id": 1,
        "username": "Tom"
    }
}
```

## 日程管理系统

### 添加日程

**[POST]**  `/schedules`

用户添加一个日程。

**请求参数**

| **参数**      | **类型** | **说明**        |
| ------------- | -------- | --------------- |
| title         | str      | 日程标题        |
| start_time    | str      | 开始时间        |
| end_time      | str      | 结束时间        |
| description   | str      | 描述            |
| lng           | float    | 地点经度        |
| lat           | float    | 地点纬度        |
| location_name | str      | 标准化地点名称  |
| amap_poi_id   | str      | 高德地图 POI ID |

**响应状态**

| **状态码** | **说明**     |
| ---------- | ------------ |
| 0          | 添加日程成功 |
| 10001      | 请求参数错误 |

**响应数据**

| **参数**    | **类型** | **说明** |
| ----------- | -------- | -------- |
| schedule_id | int      | 日程编号 |

**示例**

请求

```JSON
{
    "title": "某某会议",
    "start_time": "2026-03-05T14:30:00",
    "end_time": "2026-03-05T16:00:00",
    "description": "记得带简历",
    "lng": 116.391234,
    "lat": 39.907890,
    "location_name": "学生活动中心",
    "amap_poi_id": "B000A856LJ"
}
```

响应

```JSON
{
    "code": 0,
    "message": "添加日程成功",
    "data": {
        "schedule_id": 233
    }
}
```

### 查询日程列表

**[GET]**  `/schedules`

用户查询一个时间段内的日程列表，返回按照开始时间顺序排序的日程列表。

**请求参数**

| **参数**   | **类型** | **说明** |
| ---------- | -------- | -------- |
| start_time | str      | 开始时间 |
| end_time   | str      | 结束时间 |

**响应状态**

| **状态码** | **说明**     |
| ---------- | ------------ |
| 0          | 查询日程成功 |
| 10001      | 请求参数错误 |

**响应数据**

| **参数**  | **类型**   | **说明** |
| --------- | ---------- | -------- |
| len       | int        | 列表长度 |
| schedules | list[dict] | 日程列表 |

其中日程列表每一项格式为

| 参数          | 类型  | 说明            |
| :------------ | :---- | :-------------- |
| schedule_id   | int   | 日程编号        |
| title         | str   | 日程标题        |
| start_time    | str   | 开始时间        |
| end_time      | str   | 结束时间        |
| description   | str   | 描述            |
| lng           | float | 地点经度        |
| lat           | float | 地点纬度        |
| location_name | str   | 标准化地点名称  |
| amap_poi_id   | str   | 高德地图 POI ID |

**示例**

请求

```JSON
{
    "start_time": "2026-03-05T00:00:00",
    "end_time": "2026-03-05T23:59:00"
}
```

响应

```JSON
{
    "code": 0,

    "data": {
        "len": 2,
        "schedules": [
            {
                "schedule_id": 233,
                "title": "某某会议",
                "start_time": "2026-03-05T14:30:00",
                "end_time": "2026-03-05T16:00:00",
                "description": "记得带简历",
                "lng": 116.391234,
                "lat": 39.907890,
                "location_name": "学生活动中心",
                "amap_poi_id": "B000A856LJ"
            },
            {
                "schedule_id": 234,
                "title": "某某聚餐",
                "start_time": "2026-03-05T18:30:00",
                "end_time": "2026-03-05T19:00:00",
                "description": "无",
                "lng": 116.391234,
                "lat": 39.907890,
                "location_name": "清芬园",
```
